# Copilotカスタム設定（Instructions, Skills, Prompts）の使い方

## 概要と背景
AI（GitHub Copilot）を単なるコード生成ツールとしてではなく、「学習メンター」として自分好みに育てるために、`.github` フォルダ内に特別な設定ファイルを配置しました。

## それぞれの役割と違い

これは「会社」に例えると分かりやすいです。

### 1. `.github/copilot-instructions.md`（会社の「憲法・基本理念」）
*   **役割**: プロジェクト全体に常に適用される大前提のルール。
*   **特徴**: ここに書いたことは、どのファイルを触っていても、どんな質問をしてもAIが意識します。（例：「コードは書くな」「説明を優先しろ」）
*   **位置**: 今の場所のままで正解です。移動も削除もしません。

### 2. `.github/instructions/*.instructions.md`（部署ごとの「ローカルルール」）
*   **役割**: 特定のファイル（例: `.ts`, `.tsx`）を開いている時だけ追加で発動するルール。
*   **特徴**: 「全体ルール」に加えて、「TypeScriptを触る時はこの教え方をして」という特定の場面でのメンターのモード切り替えに使います。

### 3. `.github/skills/*/SKILL.md`（社員向けの「業務マニュアル」）
*   **役割**: 複数の手順が必要な定型作業をAIにお任せするための手順書。
*   **特徴**: AIは裏側で常にこのマニュアルの「タイトル（description）」を監視しています。ユーザーが「学んだことをまとめて（= add-knowledge）」と発言すると、AIが「あ、このマニュアルの出番だ！」と判断して、書かれた手順（①文脈を読む → ②Markdownを作る → ③保存する）を自律的に実行します。

### 4. `.github/prompts/*.prompt.md`（便利な「定型文はんこ」）
*   **役割**: いつも使う長い指示を、チャット欄から `/` で呼び出せるようにするショートカット。
*   **特徴**: `/explain-concept` のように、ユーザーが明示的にメニューから選んで実行します。

## 学んだことのまとめ
AIはただ質問に答えるだけではなく、設定ファイル（`.github`内）を充実させることで、自律的にルーチンワークをこなしたり（Skill）、特定の状況で賢く立ち回る（Instructions）ことができる。
