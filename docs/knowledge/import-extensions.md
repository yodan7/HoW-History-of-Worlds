# Import時のファイル拡張子の省略について（TS/JS vs CSS・画像）

React (Vite/Webpack等) 開発において、「なぜコンポーネント（.ts, .tsx）の拡張子は省略できて、CSS（.css）や画像（.png）は省略できないのか」という疑問についてのまとめ。

## 1. 原則：モジュールバンドラーの「自動補完」機能
開発ツール（ViteやWebpack）の裏側にいる「モジュールバンドラー（複数のファイルを1つにまとめる仕組み）」は、**デフォルトでJavaScript系のファイル拡張子を自動で探して補ってくれる機能**を持っています。

*   **自動で探す対象**: `.js`, `.jsx`, `.ts`, `.tsx`, `.json` などのプログラム関連のファイル。

つまり、`import App from "./App"` と書いた時、Viteは裏側で：
1. 「まずは `./App.js` がないか探そう」
2. 「無いな、次は `./App.ts` を探そう」
3. 「無いな、次は `./App.tsx` を探そう。あ、あった！」
という風に、**自動的にリストにある拡張子を順番に試して見つけてくれます。** これが省略できる理由です。

## 2. なぜCSSや画像は省略できないのか？
CSS、画像（PNG/SVG）、フォントなどのファイルは、**「プログラムを動かすためのソースコード」ではなく、「飾り付け（見た目・静的アセット）」** だからです。

モジュールバンドラーは、パフォーマンスを落とさないために「むやみやたらに全ての拡張子（.css, .png, .jpg...）を探し回る」設定にはなっていません。
そのため、`import "./index"` と書かれても、「`.js` や `.ts` の中身のプログラムかな？」と勘違いして探しに行き、結果的に「そんなプログラムのファイルは見つからないよ」とエラーになってしまいます。

## 結論（ルールの覚え方）
*   **プログラム系のファイル（JS, TS, JSX, TSX, JSON等のコンポーネント）**: 拡張子は **省略する** のがモダンな開発の常識（暗黙の了解）。
*   **それ以外のファイル（CSS, SCSS, 画像, フォント等アセット）**: Viteに「これはプログラムじゃないから、この拡張子をそのままファイルとして読み込んでね」と確実に伝えるために、**拡張子は絶対に明記しなければならない。**
