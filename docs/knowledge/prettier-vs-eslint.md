# PrettierとESLintの役割の違いと保存時自動実行

## どんな疑問からスタートしたか
「コメントアウトの `//` の後に半角スペースを自動で1つ入れることはできないか？」という疑問が発端。

## 解決策・学んだこと
コードを自動できれいにする仕組みには、大きく分けて**2つの異なる役割を持つツール**が存在する。

1. **Prettier（清掃員・スタイリスト）**
   - **役割**: パッと見の見た目（インデントのズレ、改行、小なり/大なりカッコなど）をただ無心で整える係。
   - **特徴**: コードの「中身のルール」には一切口出ししない。そのため、コメントの中身に勝手にスペースを入れるようなことはしない。

2. **ESLint（風紀委員・風紀チェック係）**
   - **役割**: コードがチームのコーディング規約（作法・ルール）に違反していないかをチェックし、可能なら自動で修復する係。
   - **特徴**: 「使っていない変数はエラーにする」「コメントの最初は必ずスペースを空ける (`spaced-comment`)」といった、具体的な中身のルールに介入できる。

### 設定方法
VS Codeにおいて、保存した瞬間にこれら2つのツールによる最強の連携コンボを発動させるには設定が必要。

`.vscode/settings.json` に以下を記述する。
```json
{
  "editor.formatOnSave": true, // 保存時に清掃員（Prettier）を呼ぶ
  "editor.defaultFormatter": "esbenp.prettier-vscode",
  "editor.codeActionsOnSave": {
    "source.fixAll.eslint": "explicit" // 保存時に風紀委員（ESLint）の全自動修正も呼ぶ
  }
}
```

さらに、ESLintの設定ファイル（`eslint.config.js` 等）にルールを追記する。
```javascript
  rules: {
    "spaced-comment": ["error", "always"] // コメントのあとのスペースを強制
  }
```

## 例え話（日常の比喩）
* **Prettier** は提出するレポートの**「文字のフォントや行間、余白をきれいにする人」**。
* **ESLint** はレポートの**「誤字脱字や、『です・ます』調の統一などのルール違反を直す人」**。
両者が保存時に一気に働くことで、提出物（コード）が常に完璧な状態になる！
